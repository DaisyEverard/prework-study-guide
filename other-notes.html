Other Notes

Development Jargon:

  • User stories The type of user, what they want from an app, why they need it (what problem are they solving?)
  • Acceptance criteria Is it done? List of thing app needs to be finished

Computational Thinking:

  • Decomposition Breaking into small tasks. write pseudocode as steps in comments
  • Pattern recognition repeating similar subtasks
  • Abstraction Ignore irrelevant details
  • Algorithms ordering, event triggers, conditions, debugging
  • Debugging finding mistakes and gaps

CLI (Command line interface):

  • pwd (print working directory) where you are now
  • ls list files in working directory
  • cd __ change directory to __
  • cd .. change back one directory level
  • mkdir make a new directory
  • touch file-name.txt create new text file
  • start __ open file
  • code . open vs code
