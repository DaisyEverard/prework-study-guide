Git

Jargon:

  • Git version control software. Allows users to make changes simultaneously
  • Repo (repository) name of project folder + revision history
  • Branch space for changes that doesn't affect live site
  • Markdown Plain text that converts to HTML easily. stored in .md files
  • Commit Changes adding, editing, or deleting files in repository
  • Pull Request request to merge changes from on branch with another. Add explanation comments and request reviewers
  • Merge publish/deploy changes to live site
  • Issue a way to report a new task or bug to fix. Tracked on project board.

Creating new branch

  1. git status check which branch you're on
  2. git checkout -b feature/starter-code create new branch
  3. git status

Merging branch

  1. git status
  2. git add -A
  3. git commit -m
  4. git pull origin main sync local branch with repo version
  5. git push origin feature/starter-code push changes to github branch
  6. Go to git hub and follow instructions

What those git elements mean:

git Precedes all git statements

checkout Moves working branch to new branch

-b Create new branch

Feature/starter-code Name of new branch. Freature prefix shows change is adding new feature

add add modifications in CWB to staging area

-A add all changes

-m associate message

pull reveive remote branch's modifications into local environment

origin source of info to pull

main the name of a branch in the repo
